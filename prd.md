# 프로젝트 개요

- 프로젝트명
  - 생기부 개인정보 익명화 도구

- 프로젝트 목적
  - Excel 기반 학교생활기록부 파일에서 학생 이름 및 학교명을 자동 탐지하고 익명화하여 개인정보 보호 및 외부 제출 작업을 효율화한다.

- 핵심 문제
  - 생기부 문서 내 개인정보를 수작업으로 수정하는 과정이 반복적이고 시간이 많이 소요됨
  - 수작업 수정 시 누락 및 오타 위험 존재
  - 완전 자동 삭제 방식은 오탐 위험이 있어 검수 과정 필요

- 타겟 사용자
  - 중학교 및 학교 업무 담당자
  - 생활기록부 검수 및 제출 업무 사용자

- 핵심 가치 제안
  - 개인정보 익명화 자동화
  - 검수 기반 안전 처리
  - 원본 유지 중심 안전 구조
  - 설치 없이 실행 가능한 Windows 업무 툴

# 기술 설계

## 기술 스택

### 언어
- Python 3.11+

### GUI
- PySide6

### Excel 처리
- openpyxl
- pandas

### 배포
- PyInstaller

---

## 선택 이유

### Python
장점:
- Excel 자동화 생태계 강력
- 빠른 MVP 개발 가능
- 유지보수 용이
- AI 협업 친화적

대안:
- C#
- Electron

선택하지 않은 이유:
- 초기 개발 속도 저하
- 유지보수 복잡도 증가 가능

---

### PySide6
장점:
- 현대적 GUI 가능
- 테이블 UI 강력
- Windows 호환 우수
- 유지보수 용이

대안:
- Tkinter
- PyQt

선택 이유:
- Tkinter보다 UI 품질 우수
- PyQt 대비 라이선스 부담 감소

---

### openpyxl
장점:
- xlsx 처리 안정적
- 셀 단위 수정 가능
- 서식 유지 가능성 높음

## 데이터 구조

### 개인정보 매핑 구조

```python
{
    "김민수": "학생1",
    "이서연": "학생2",
    "서울중학교": "학교A"
}
```

---

### 탐지 결과 구조

```python
{
    "sheet": "Sheet1",
    "cell": "B12",
    "original": "김민수",
    "replacement": "학생1",
    "approved": True
}
```
