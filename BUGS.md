## 2. 잠재적 리스크 및 예외 상황 관리 (Risks & Exceptions)

| 위험 요소 / 예외 상황 | 발생 원인 | 예상 영향 | 대응 및 조치 계획 |
| :--- | :--- | :--- | :--- |
| **Excel 병합 셀 (Merged Cells)** | 시트 내 병합된 셀이 존재할 때 치환 시도 | - 치환 누락 발생<br>- 서식 손상 | 병합 셀 여부를 감지하고, 대표 셀(좌상단 셀)에만 값을 설정하도록 `AnonymizeDetector`에서 병합 셀 처리를 통합 수행. |
| **숨김 시트 (Hidden Sheets)** | Excel 내 사용자가 보지 않는 숨김 시트 존재 | - 불필요한 연산 낭비<br>- 숨겨진 개인정보 누락 | `openpyxl` 스캔 시 시트의 가시성 속성(`sheet.sheet_state`)을 체크하여 활성 시트만 탐지하도록 필터링 제공. |
| **수식 포함 셀 (Formula Cells)** | 셀 내용이 일반 텍스트가 아닌 `=SUM(...)` 등의 수식인 경우 | - 수식이 문자열 치환되어 깨짐<br>- 값 계산 오류 | 수식이 들어있는 셀(첫 글자가 `=`인 셀)은 치환 대상에서 제외하고 원본 수식을 유지. 필요시 경고 로그 기록. |
| **대용량 파일 성능 저하** | 행/열이 매우 많은 Excel 파일 로드 및 스캔 | - UI 메인 스레드 정지(Freeze)<br>- 메모리 과소비 | 1. `QThread`를 사용해 백그라운드에서 Excel을 스캔하도록 설계.<br>2. 렌더링 성능 향상을 위해 테이블은 Virtual Scroll 또는 부분 페이징 기법 고려. |
| **한글 및 특수기호 파일 경로** | Windows 환경에서 한글 폴더명/파일명 사용 | - 파일 로드 실패<br>- 인코딩 에러 | Python의 `pathlib.Path`를 사용하여 경로를 객체화하고 OS 독립적인 UTF-8 경로 전달 수행. |
| **동작 중 비정상 종료 (Crash)** | 저장 또는 변환 과정 중 프로그램 비정상 종료 | - 원본 Excel 파일 훼손 | **Safe Save 4단계 원칙 준수**:<br>1. temp 폴더에 임시 파일 생성 및 쓰기<br>2. 저장 성공 검증<br>3. 최종 output 경로에 덮어쓰기 없이 새 이름으로 파일 이동<br>4. 원본 파일은 읽기 모드(`read_only=True` 또는 단순 load)로만 접근하여 보호. |
