# STATUS (현재 개발 상태)

이 문서는 생기부 개인정보 익명화 도구 프로젝트의 **현재 구현 상태**, **다음 작업 계획**, **알려진 이슈** 및 **기술 부채**를 관리합니다.

---

## 1. 현재 구현율 및 진척 상황

* **현재 상태**: **v5.0 배포 준비 완료 (HWP/HWPX 다중 포맷 확장 및 아키텍처 개편 완료)**
* **전체 진척률**: **100%**
* **개발 현황**:
  - 기존 엑셀 전용 구조에서 `BaseProcessor` 인터페이스 기반의 파일 포맷 다형성 아키텍처로 개편을 마쳤습니다.
  - HWPX 직접 XML 파싱/치환(`HwpxProcessor`) 및 HWP 직접 COM 텍스트 스캔(`HwpProcessor`) 모듈을 탑재하였고, 파일 선택 다이얼로그와 UI 라벨 텍스트를 범용화하여 자연스럽게 연동했습니다.
  - 개인정보 탐지 시 전후 문맥을 확인할 수 있는 `context_preview` (40~60자) 필드를 추가하고 UI `PreviewTable`을 6컬럼으로 확장 완료했습니다.
  - PySide6 내장 `QSystemTrayIcon` 기반의 스레드 안전한 바탕화면 트레이 알림 송출 시스템(`NotificationHelper`)을 구축하여 작업 편의성을 극대화했습니다.

* **현재 발견된 활성 버그는 없습니다.**
* HWP 포맷의 가명화 치환 기능은 COM 제어의 서식 뒤틀림 리스크 격리를 위해 PoC 단계로 제한해두었습니다 (스캔 탐색만 활성화).

---

## 4. Technical Debt (기술 부채)

1. **AppState 직접 수정 구조**
   - 현재 `AppController` 내에서 `self.state.selected_files = ...` 형식으로 상태 필드를 직접 덮어쓰고 있습니다. 프로젝트 규모가 확장될 경우 상태 추적이 어려울 수 있으므로, 상태 변경용 액션 패턴이나 캡슐화 메서드를 도입하는 개선이 필요할 수 있습니다.
2. **복구 파일 이력 제공 부재**
   - 앱 실행 시 `cleanup_temp_folder()`를 무조건 호출해 temp 폴더를 비우도록 설계되어 있습니다. 비정상 종료 시 사용자가 복구용 temp 파일을 인지하고 활용할 수 있는 복구 정보 이력 화면이 장기적으로 필요할 수 있습니다.
3. **HWP OLE 치환 기능 미완성 (PoC 격리)**
   - HWP 직접 치환 시 오프셋 및 폰트 서식 깨짐 현상이 우려되어 현재 비활성화 상태입니다. 안정적인 쓰기 인터페이스 구현이 기술 부채로 남아 있습니다.
